"Make a choice" is difficult, especially there are lots of web technologies, like: PHP, Ruby, JAVA, JavaScript, ASP, and so on. You are just blind since each seems good, and each needs time to handle.

I present you 'Node.js'. for the following 3 reasons:

  • It makes JavaScript as a server side language,
  • It is lightweight and very fast to establish a web server,
  • Its non-blocking I/O feature makes easier to scale

    • Here is a description of Node.js from its site:
    Node.jsÂ® is a platform built on Chrome's JavaScript runtime for easily building fast, scalable network applications. Node.js uses an event-driven, non-blocking I/O model that makes it lightweight and efficient, perfect for data-intensive real-time applications that run across distributed devices.


    And look how fast and easy it establishes a web server:

    var http = require('http');
http.createServer(function (req, res) {
  res.writeHead(200, {'Content-Type': 'text/plain'});
  res.end('Hello World\n');
}).listen(1337, '127.0.0.1');
console.log('Server running at http://127.0.0.1:1337/');

    Node.js is the very raw material, it helps you to solve some fundamental technique issues, like: TCP/IP, a server-side coding language based on JavaScript. But if you want to establish a http server, you still have a lot of things to do. You need a web application framework for Node.js that provides you HTTP fundamentals. The framework is out-of-box: 'Express.js'

    'Express.js' provides:

  • A web application framework,
  • Http request api,
  • Http response api,
  • Http router api,
  • Middleware supports to allow you inject your codes into http requests and responses.

    • We are now having the coding language and the framework technologies. We can now develop our web application...... with a text editor? Of course, but if you want develop efficiently, you need an IDE. Eclipse is not my cup of tea, there is a good IDE for JavaScript, the smartest JavaScript IDE: 'WebStorm'.


    Are HTML5, CSS, and JavaScript enough for the client side coding? Yes, but very raw. If you do your work based on these 3 raw technologies,you won't get your web application running after 1 year. Thanks for the open-source community, you have 'Bootstrap' for a good appearance, and 'AngularJS' for dynamic views.
    Both 'Bootstrap'(provided by Twitter) and 'AngularJS'(provided by Google) are big frameworks which need time to handle. They are not easier than Node.js or Express.js. Hope you already have some knowledge on 'HTTP, HTML, CSS, and JavaScript'.

    We can use 'WebStorm' to do server-side testing and debug, but what about the client-side testing and debug tools? My suggestion is using Chrome Developer Tools. It is good enough: doing CSS adjustment in a WYSIWYG way, and debug JavaScript on-hands.


    One more thing: how to persist the data?
    This is not a question 5 years ago, but a difficult choice nowadays. Since the NoSQL trends, we now have a lot of DB technologies:

  • 'MongoDB', a Document-Oriented storage in JSON-style. Seems to be a good companion with Node.js.
  • 'Memcached', a Key-Value in-memory storage caching system, but not persists data.
  • 'Redis', a Key-Value in-memory storage system which persists data.
  • 'MySQL', The most popular open-source RDS.

    • What are my choices? It took me 1 week to make the decision: 'MySql' and 'Memcached'. 'MongoDB' is indeed a good companion with Node.js with its JSON-stype storage(without data conversion in Node.js). But the schema change is not smooth, and loses data! So I design a new data schema based on RDS which is described in Chapter 04.
    'Redis' is too big compared with 'Memcached', I need a lightweight cache system which data persistence is not necessarily.

  • 'Mocha', JavaScript unit testing framework.
  • 'node-mysql', A pure node.js JavaScript Client implementing the MySql protocol.

  • 'AngularUI', The companion suites to AngularJS framework.
  • 'UnderScore.js', A JavaScript library that provides a whole mess of useful functional programming helpers without extending any built-in objects.
